## COA-012 — Evaluation Scenario Families

### Objective

Expand deterministic evaluation coverage from a single sample-question file into scenario families.

### Required Changes

Create or update:

```text
examples/evaluation/guardrails.json
examples/evaluation/context_gaps.json
examples/evaluation/assumptions.json
examples/evaluation/calculation_readiness.json
src/carbonops_assistant/evaluation/cases.py
tests/test_evaluation_cases.py
```

### Scenario Family Requirements

Add small JSON files with cases grouped by scenario type:

- guardrails: risky claim, certification/compliance wording, unsupported completeness claims
- context gaps: vague calculation, missing factor, missing boundary, ambiguous data source
- assumptions: questions requiring limitations or stated assumptions
- calculation readiness: questions that include enough activity/factor/unit context to be answered by future deterministic workflow

Each case should include:

- `id`
- `question`
- `expected_status`
- `scenario`
- `notes`

Keep scenario files small: 4–8 cases each.

### Implementation Requirements

- Extend case loading if needed to support scenario metadata.
- Preserve compatibility with existing sample questions if practical.
- Validate `expected_status` against the response contract.
- Validate required fields.
- Do not add LLM evaluation.

### Tests

Add or update tests for:

- each scenario file loads successfully
- missing `scenario` is rejected for scenario-family files if implemented as required
- invalid expected status is rejected
- legacy sample question loading still works if preserved

### Completion Criteria

- scenario files exist and load deterministically
- `python -m pytest -q` passes
- commit created

## COA-014 — CLI Evaluation Command

### Objective

Expose deterministic evaluation running through the local CLI.

### Required Changes

Update:

```text
src/carbonops_assistant/cli.py
tests/test_cli.py
```

Create if useful:

```text
tests/test_evaluation_cli.py
```

### CLI Requirements

Support a minimal command such as:

```bash
python -m carbonops_assistant.cli status
python -m carbonops_assistant.cli evaluate examples/evaluation/guardrails.json
python -m carbonops_assistant.cli evaluate examples/evaluation/guardrails.json --json
```

Behavior:

- `status` remains conservative and local.
- `evaluate` loads cases, runs deterministic evaluator, and prints summary.
- Text output should include total, passed, failed, success.
- `--json` should print deterministic JSON report.
- Exit code:
  - `0` when all cases pass
  - non-zero when any case fails or case file is invalid

Do not add external calls.

### Tests

Add tests for:

- status command still works
- evaluate text output works
- evaluate JSON output works and is parseable
- failing case returns non-zero code

### Completion Criteria

- CLI evaluation is local and deterministic
- `python -m pytest -q` passes
- commit created

# CarbonOps Assistant — Codex Night Run — 2026-04-30

## Purpose

This run initializes CarbonOps Assistant as a small, credible, pre-alpha AI/domain assistant project.

The goal is not to build a production system. The goal is to establish a clean technical foundation that can later support:

- domain-specific carbon/emissions assistant behavior
- deterministic emissions calculation helpers
- structured response contracts
- guardrails for unsupported claims
- evaluation and benchmark workflows
- future Hugging Face dataset/model evidence

## Global Constraints

Follow these constraints for the entire run.

### Allowed

- Create a minimal Python package structure.
- Add `pyproject.toml`.
- Add small deterministic Python modules.
- Add unit tests.
- Add documentation.
- Add a test-only GitHub Actions workflow.
- Commit each completed task separately.
- Push only if validation passes.

### Forbidden

Do not:

- add deployment automation
- publish to PyPI
- add Docker registry workflows
- add cloud infrastructure
- add real secrets or tokens
- add external paid service dependencies
- claim production readiness
- claim regulatory compliance
- claim certification
- claim full emissions accounting coverage
- add complex UI
- add authentication or multi-tenancy
- over-engineer provider/plugin architectures

### Language and Tone

Documentation must be factual and conservative.

Avoid phrases such as:

- production-ready
- enterprise-grade
- certified
- compliant
- fully automated
- regulatory approved
- complete emissions platform

Use:

- pre-alpha
- experimental
- local development
- baseline
- planned
- not production-ready

## Validation Rules

After each task:

```bash
python -m pytest -q
git status
```

If tests fail:

1. Fix the failure if the cause is clearly within the current task.
2. Re-run tests.
3. If still failing, stop and report.
4. Do not continue to the next task.

Before final push, run:

```bash
python -m pytest -q
grep -RniE "production-ready|enterprise-grade|certified|compliant|fully automated|regulatory approved|docker login|pypi|secret|token|password" README.md docs .github src tests pyproject.toml || true
git status
```

If risky language appears, inspect it. It may be acceptable only when used in a forbidden/non-goal/disclaimer context.

## Commit Rules

Use small commits.

Suggested commit message format:

```text
COA-001 Add Python package baseline
COA-002 Improve project documentation baseline
COA-003 Add assistant response contract and guardrails
COA-004 Add emissions calculation helpers
COA-005 Add baseline tests and CI validation
```

Do not squash.

Do not commit broken tests.

Push only if all completed commits pass validation.

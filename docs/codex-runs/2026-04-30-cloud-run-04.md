## COA-016 — Evaluation Report Snapshot Helpers

### Objective

Add deterministic helpers for saving/loading evaluation summary JSON so reports can be compared over time.

### Required Changes

Create or update:

```text
src/carbonops_assistant/evaluation/snapshot.py
tests/test_evaluation_snapshot.py
```

Update if needed:

```text
src/carbonops_assistant/evaluation/report.py
```

### Implementation Requirements

Implement standard-library-only helpers such as:

- `save_report_snapshot(report, path)`
- `load_report_snapshot(path)`
- optional `normalize_report_payload(payload)` if useful

Requirements:

- JSON output/input must be deterministic.
- Loaded snapshot should expose fields needed for diffing:
  - total cases
  - passed count
  - failed count
  - success
  - results by scenario
  - failed cases
- Do not add timestamp fields by default; timestamps make deterministic tests harder.
- Do not write files outside the path provided by caller.

### Tests

Add tests for:

- saving and loading a report snapshot
- deterministic JSON content
- missing required snapshot fields produces a clear error
- snapshot payload preserves failed case details

### Completion Criteria

- snapshot helpers exist
- deterministic tests pass
- `python -m pytest -q` passes
- commit created

## COA-017 — Evaluation Report Diff Tooling

### Objective

Add deterministic diff tooling for two evaluation summary snapshots.

### Required Changes

Create:

```text
src/carbonops_assistant/evaluation/diff.py
tests/test_evaluation_diff.py
```

Update if useful:

```text
src/carbonops_assistant/cli.py
tests/test_cli.py
```

### Diff Requirements

Implement a small diff object/function comparing two loaded snapshot payloads.

The diff should include:

- previous total / current total
- previous passed / current passed
- previous failed / current failed
- newly failing case IDs
- newly passing case IDs
- unchanged failing case IDs
- success changed boolean
- compact deterministic JSON export

Keep the logic status/case-id based. Do not compare natural-language answer text yet unless already available and stable.

### Optional CLI

If simple and well-tested, add:

```bash
python -m carbonops_assistant.cli compare-evaluations previous.json current.json
python -m carbonops_assistant.cli compare-evaluations previous.json current.json --json
```

Behavior:

- text output should be compact and deterministic
- `--json` should print deterministic JSON diff
- exit code can be `0` unless input is invalid; do not fail simply because evaluation got worse unless docs clearly specify that behavior

### Tests

Add tests for:

- newly failing cases
- newly passing cases
- unchanged failing cases
- success changed
- deterministic JSON output
- CLI behavior if implemented

### Completion Criteria

- diff tooling exists
- no external dependencies
- `python -m pytest -q` passes
- commit created
